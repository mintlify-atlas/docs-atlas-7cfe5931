## Overview

The Bet Model defines data structures for tracked bets and arbitrage opportunities in the PROPPR platform.

## Tracked Bet Document

### Schema

<ParamField path="bet_id" type="string" required>
  Unique bet tracking ID (ObjectId string)
</ParamField>

<ParamField path="alert_id" type="string">
  Reference to original alert
</ParamField>

<ParamField path="tracked_at" type="datetime" required>
  Timestamp when bet was tracked
</ParamField>

<ParamField path="player_name" type="string">
  Player name (Player Bot only)
</ParamField>

<ParamField path="player_id" type="integer">
  Player ID (Player Bot only)
</ParamField>

<ParamField path="team" type="string">
  Team name
</ParamField>

<ParamField path="team_id" type="integer">
  Team ID
</ParamField>

<ParamField path="market" type="string" required>
  Betting market (e.g., "Player Shots On Target")
</ParamField>

<ParamField path="match" type="string" required>
  Match name (e.g., "Liverpool vs Man City")
</ParamField>

<ParamField path="date" type="string">
  Match date (YYYY-MM-DD)
</ParamField>

<ParamField path="time" type="string">
  Match time (HH:MM)
</ParamField>

<ParamField path="threshold" type="float">
  Market threshold (e.g., 1.5)
</ParamField>

<ParamField path="market_direction" type="string">
  "over" or "under"
</ParamField>

<ParamField path="bookmaker" type="string">
  Bookmaker name
</ParamField>

<ParamField path="odds" type="float" required>
  Decimal odds
</ParamField>

<ParamField path="actual_stake" type="float" required>
  Actual stake in currency
</ParamField>

<ParamField path="unit_size" type="float" required>
  User's unit size
</ParamField>

<ParamField path="units_staked" type="float" required>
  Stake in units
</ParamField>

<ParamField path="status" type="string" default="pending">
  "pending", "won", "lost", "refund", "half_win", "half_loss"
</ParamField>

<ParamField path="manual_override" type="boolean" default="false">
  If true, bet was manually graded (protected from auto-grading)
</ParamField>

</ParamField>

<ParamField path="profit_loss" type="float" default="0.0">
  Profit or loss amount
</ParamField>

<ParamField path="fixture_id" type="integer">
  Fixture/match ID
</ParamField>

<ParamField path="settled_at" type="datetime">
  Timestamp when bet was settled
</ParamField>

<ParamField path="actual_value" type="float">
  Actual statistic value from match
</ParamField>

<ParamField path="result_tracking" type="object">
  Result tracking metadata for auto-grading
</ParamField>

## Status Enums

### Bet Status

- `pending` - Awaiting result
- `won` - Bet won
- `lost` - Bet lost
- `refund` - Bet refunded (push)
- `half_win` - Half stake won (Asian handicap)
- `half_loss` - Half stake lost (Asian handicap)

### Market Direction

- `over` - Over threshold
- `under` - Under threshold

### Bet Sides

- `over` - Over (totals)
- `under` - Under (totals)
- `home` - Home team
- `away` - Away team
- `draw` - Draw
- `yes` - Yes
- `no` - No
