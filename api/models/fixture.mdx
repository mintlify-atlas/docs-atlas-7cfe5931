## Overview

The Fixture Model defines data structures for match/event data, including participants, statistics, and odds from multiple bookmakers.

## FotMob Fixture Structure

### Core Fixture Document

<ParamField path="general.matchId" type="integer" required>
  Unique match ID
</ParamField>

<ParamField path="general.matchName" type="string" required>
  Match name (e.g., "Liverpool vs Man City")
</ParamField>

<ParamField path="general.matchRound" type="string">
  Competition round
</ParamField>

<ParamField path="general.leagueId" type="integer">
  League/competition ID
</ParamField>

<ParamField path="general.leagueName" type="string">
  League name
</ParamField>

<ParamField path="general.parentLeagueId" type="integer">
  Parent league ID (for sub-competitions)
</ParamField>

## Fixture Status

### Status Values

- `NS` - Not Started
- `LIVE` - In Progress
- `HT` - Half Time
- `FT` - Full Time
- `AET` - After Extra Time
- `PEN` - Penalties
- `PST` - Postponed
- `CANC` - Cancelled
- `ABD` - Abandoned
- `SUSP` - Suspended

### Check if Finished

```python
def is_fixture_finished(fixture_data: dict) -> bool:
    """Check if fixture has finished"""
    status = fixture_data.get('header', {}).get('status', {})
    return status.get('finished', False)
```

### Check if Started

```python
def is_fixture_started(fixture_data: dict) -> bool:
    """Check if fixture has started"""
    status = fixture_data.get('header', {}).get('status', {})
    return status.get('started', False)
```
